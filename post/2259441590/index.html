Problem B

同样的贪心 + 分讨。

第一种情况是去掉上界，最为简单，将剩余的全给利润最大的即可。（但我一开始漏掉了）

第二种较为复杂，枚举去掉下界的作物的下标 iii，此时剩余的时间 rest←m−∑l+lirest \gets m - \sum{l} + l_irest←m−∑l+li​，去掉之后的总利润 cur←∑l×w−li×wicur \gets \sum {l \times w} - l_i \times w_icur←∑l×w−li​×wi​。

根据贪心不难想出剩余的优先给大的，因此从大到小排序，定义数组 precntprecntprecnt 表示前 iii 个可以添加剩余的个数，presumpresumpresum 表示装满前 iii 个剩余个数所增加的利润值。有：

precnti=precnti−1+ri−lipresumi=presumi−1+(ri−li)×wiprecnt_i = precnt_{i - 1} + r_i - l_i\\ presum_i = presum_{i - 1} + (r_i - l_i) \times w_i precnti​=precnti−1​+ri​−li​presumi​=presumi−1​+(ri​−li​)×wi​

显然单调，二分找到最后一个小于 restrestrest 的 precntprecntprecnt 的下标 idxidxidx，答案即为：

cur+presumidx+(rest−precntidx)×wicur + presum_{idx} + (rest - precnt_{idx}) \times w_i cur+presumidx​+(rest−precntidx​)×wi​

Problem C

真正意义上的第一道思维题。

将一个序列从第 iii 处断开，那么这个 gcdgcdgcd 的值为 gcd⁡(prei,sufi+1)\gcd(pre_i, suf_{i + 1})gcd(prei​,sufi+1​)，其中 preprepre 和 sufsufsuf 分别是序列的前缀和后缀 gcd⁡\gcdgcd 值，pre0=sufn+1=0pre_0 = suf_{n + 1} = 0pre0​=sufn+1​=0。

扩展到本题上仍然适用，答案 ansansans 为：

max⁡1≤l≤r≤n(gcd⁡(gcd⁡(prel−1,sufr+1),gcd⁡l≤i≤rai+k))\max_{1 \le l \le r \le n} \left(\gcd \left(\gcd(pre_{l - 1}, suf_{r + 1}), \gcd_{l \le i \le r} a_i + k \right) \right ) 1≤l≤r≤nmax​(gcd(gcd(prel−1​,sufr+1​),l≤i≤rgcd​ai​+k))

区间枚举的时间复杂度难以承受，但是考虑一种贪心，枚举左端点 LLL，找到最右的端点 RRR 使得 preL−1=gcd⁡L≤i≤Rai+kpre_{L - 1} = \gcd_{L \le i \le R} a_i + kpreL−1​=gcdL≤i≤R​ai​+k，如果此时的 sufR+1=preL−1suf_{R + 1} = pre_{L - 1}sufR+1​=preL−1​，那么说明此时的答案成立。看似最坏的时间复杂度没有变化，其实已经可以通过了。

这是因为选择的左端点 LLL 只在它即将变化时才遍历，由于 gcd⁡\gcdgcd 的性质，前缀最大公约数中不同的值最多只会有 log⁡W\log WlogW 个（每一个 preipre_iprei​ 都是 prei−1pre_{i - 1}prei−1​ 的因数），因此枚举的时间复杂度仅为 O(log⁡W)O(\log W)O(logW) 次，总的时间复杂度为 O(n×log⁡W)O(n \times \log W)O(n×logW)​，可以通过。

Problem D

没有完全想出来。

一开始想的是按边所属的公司分类，定义一组 set，setiset_iseti​ 存储所属公司是 iii 的边的下标，将边加进去当且仅当这条边的一个起点刚刚被达到，而另一个起点没有达到，删去这条边当两个点都被到达过了，按照边的权值从小到大进行排序。

这样写的复杂度可能会超标，因为每当确立一个节点，就要遍历它相邻的所有节点，时间复杂度可以达到 O(km)O(km)O(km)，同时不能很好的处理同时走过两条边以上的情况。

正解的做法其实十分接近，将 set 换成堆来做，对于每一对 aia_iai​ 和 bib_ibi​，我们都将 heap[a[i]] 中边权小于 bib_ibi​ 的加入到一个堆中，这个堆用来跑 Dijkstra。比较之下我想出的做法更像暴力 BFS。

Problem H

代码好写，但是第一步就没想出来。

策略：一直随机到直到小于等于某个值 ccc 后一直减。

题解的策略是这样的，但我死活也想不出来。

设这个值为 ccc，值域为 TTT，那么每一次随机成立的概率 p=cTp = \frac{c}{T}p=Tc​，那么操作次数 cntcntcnt 有：

cnt=c−12+1+(1−p)1+(1−p)2+(1−p)3+…=c−12+1p\begin{align} cnt &= \frac{c - 1}{2} + 1 + (1 - p)^1 + (1 - p)^2 + (1 - p)^ 3 + \dots \\ &= \frac{c - 1}{2} + \frac{1}{p} \end{align} cnt​=2c−1​+1+(1−p)1+(1−p)2+(1−p)3+…=2c−1​+p1​​​

至于为什么成立，将 ∑i=1∞(1−p)i\sum_{i = 1}^{\infty} (1 - p) ^ i∑i=1∞​(1−p)i 换元即可证明。

由于 cntcntcnt 满足

cnt≥2c−12×1pcnt \ge 2 \sqrt{ \frac{c - 1}{2} \times \frac{1}{p} } cnt≥22c−1​×p1​​

因此 ccc 的取值介于 ⌊2T⌋\left \lfloor \sqrt{2T} \right \rfloor⌊2T​⌋ 和 ⌈2T⌉\left \lceil \sqrt{2T} \right \rceil⌈2T​⌉ 之间，分数运算即可。

Problem J

一开始看错题导致的，本以为筹码会从输的人给赢的人，这样子总的非平局次数必然不超过 O(log⁡n)O(\log n)O(logn) 次，直接深搜即可。

但是写完之后才发现筹码会不断减少，这就意味这一旦有一个人筹码很小并且一直赢，这样的状态树会退化成一条链（严谨来说是一条毛毛虫或者想一条链的二叉树）。

因为是一种链形的结构，自然可以区间加速，但是没有找到很巧妙的做法。

看了题解，题解说的是直接辗转相减变成辗转相除，和我想得差不多，其实在原来的代码上面改一下即可，例如当且递归的 x,yx,yx,y 有 x>yx \gt yx>y，直接加速到 x≤yx \le yx≤y，并直接用快速幂算出中间的概率，最后递归的时候返回即可，对于 x<yx \lt yx<y 同理。需要注意的是当 x>yx \gt yx>y 时的加速过程中，除了一直输，赢一次就算做贡献，直接容斥即可，具体操作代码有。

Problem M

简单思维，一堆细节处理，而且样例弱的一批。

先断环为链，用双指针进行两次处理，第一次从 111 开始，第二次从 222 开始，向右扩展的同时判断剩余的三种珠子个数，，注意到下标每次增加 222。

说一下注意的细节：指针移动的时候注意边界（下标一次性加 222），答案大于环的长度时取一个 min⁡\minmin。
