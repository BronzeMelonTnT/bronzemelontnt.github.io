复杂度

查询两点之间是否连通：O(m)O(m)O(m)。

遍历一个点所有出边：O(m)O(m)O(m)。

遍历整张图：O(n×m)O(n \times m)O(n×m)​。

适用于最小生成树 Kruskal 算法。

邻接矩阵

做法

在点的个数较小的情况下，建立一个二维数组 g[N][N]，让 g[u][v] 存储从 uuu 点到达 vvv 点的权值。

复杂度

查询两点之间是否连通：O(1)O(1)O(1)。

遍历一个点所有出边：O(n)O(n)O(n)。

遍历整张图：O(n2)O(n ^ 2)O(n2)​。

空间复杂度：O(n2)O(n^2)O(n2)。

适用于 Floyd 算法 等点数小的稠密图。

邻接表

通过 C++ 的 STL 容器 vector 定义一个支持动态增加元素的二维数组，从而减少储存和遍历非边的时间和空间。

遍历一个节点出边的时间复杂度依情况而定。

适用于一切的图，但是由于 vector 的常数较大，实际运行效率不如链式前向星（但是好理解）。

链式前向星

首先定义一个表头数组 h[N]，使得图上所有的点都对应一个表头。我们让每一条边都有一个编号 idx ，接着定义三个数组 e[N]，ne[N]，w[N]。

对于 e[idx]，表示第 idxidxidx 条边的出度。

对于 w[idx]，表示第 idxidxidx 条边的权值。

对于 ne[idx]，表示第 idxidxidx 条边的入度所对应的下一条边的编号。

每一次增加了一条边后，idxidxidx 也要相应的增加。
