多项式的两种表示方法

第一种也是最常见的一种方法：系数表示法。这种方式直接表示出了多项式 f(x)f(x)f(x) 的每一项系数，因此任意一点的点值我们都可以在线性时间内求出。同样，这种方式也方便进行计算机的存储，让次数和系数一一对应。

第二种同样十分重要：点值表示法。我们选取多个不同的 xix_ixi​，将 xix_ixi​ 带入进 f(x)f(x)f(x) 得出对应的点值 (x0,y0),(x1,y1),(x2,y2),…(x_0, y_0), (x_1, y_1), (x_2, y_2), \dots(x0​,y0​),(x1​,y1​),(x2​,y2​),…。问题在于：应该选取多少个不同的 xix_ixi​ 才可以唯一的确定一个多项式 fff 呢？

至于如何确定，这就相当于解一个 nnn 元线性方程组，一共多少组才能确定唯一解。

先给出结论：nnn 个点值唯一确定的多项式的度数为 n−1n - 1n−1。

快速傅里叶变换（FFT）

如果想从纯数学知识或者通信方面了解 FFT，请尝试先了解离散型傅里叶变换（DFT，Discrete Fourier Transform）的概念，FFT 是 DFT 的改进和转化（等我会了再说）。

而如果像我一样从计算机算法方面，只需要知道 FFT 可以用来优化多项式乘法就行了。

给定多项式 fff 和 ggg，求出多项式 h=f×gh = f \times gh=f×g 的各个项的系数。朴素做法很简单，时间复杂度为 O(n2)\mathcal{O}(n ^ 2)O(n2) 的。而快速傅里叶变换算法让这个时间复杂度降为了 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn)。

算法方向

前文提到过，系数表示法下的多项式乘法只能做到平方级别，然而点值表示法可以线性完成。这启发我们，如果我们对于多项式 f,gf,gf,g，花费一定代价分别求出 deg⁡f+deg⁡g\deg f + \deg gdegf+degg 个点值对，将这些点对对应相乘，此时乘法的时间复杂度就降到了线性了。但是这种考虑不够充分，因为我们要预先对两个多项式分别求 deg⁡f+deg⁡g+1\deg f + \deg g + 1degf+degg+1 次求值，然而每一次求值都花费 O(n)\mathcal{O}(n)O(n) 的时间，因此时间复杂度仍然为 O(n2)\mathcal{O}(n ^ 2)O(n2)。

FNTT 的优势

在了解了 FFT 之后，不难发现虽然算法的时间复杂度达到了 O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn)，但是在具体实现的时候用到了复数和 double 类型。这显然是非常不好的，在 nnn​ 非常大或者值域很大时，会产生精度错误，并且大量的复数乘法给算法产生了大常数。

那么有什么其它的东西可以代替复数和单位根，精度高，运算快，满足竞赛需求，并且允许我们像类似复数运算那样完成 FFT 算法呢？

有的，兄弟有的，它叫做原根，被广泛用于快速数论变换（Fast Number Theoretic Transform）。

原根

这里再提一下原根是什么：在模数域 Zm\mathbb{Z}_mZm​ 下，如果有一个数 g∈Zmg \in \mathbb{Z}_mg∈Zm​ 满足 g⊥mg \perp mg⊥m 且 δm(g)=φ(m)\delta_m(g) = \varphi(m)δm​(g)=φ(m)，我们将 ggg 称之为 mmm​​ 的原根。

我们定义 ωn:=gφ(m)/n\omega_n := g^{\varphi(m) / n}ωn​:=gφ(m)/n 定义模数域下的单位根。

那原根为什么可以用来替代复数域中的单位根呢，它有如下几个性质：

  • ωn\omega_nωn​ 的最小正周期为 nnn，它可以生成一共 nnn 个模意义下互不相同的数。

  • ωn\omega_nωn​ 必然存在逆元，因为 ωn⊥m\omega_n \perp mωn​⊥m，ωnk\omega_n^kωnk​ 的逆元为 ωn−k\omega_n^{-k}ωn−k​。

  • ωn\omega_nωn​ 具有周期性质，即 ωnk=ωnk+n\omega_n^k = \omega_n^{k + n}ωnk​=ωnk+n​，因此 ωn−k=ωnn−k\omega_n^{-k} = \omega_n^{n - k}ωn−k​=ωnn−k​。

  • 具有迭代传递性质，即 ωnk=ω2n2k\omega_n^k = \omega_{2n}^{2k}ωnk​=ω2n2k​。

  • 中心对称性质：

    ωnn≡ωnn/2ωnn/2≡1(modm)⇒ωnn/2≡−1(modm)⇒ωnk+n/2≡−ωnk(modm)⇒ωnk=−ωnk+n/2\omega_n^n \equiv \omega_n^{n / 2}\omega_n^{n / 2} \equiv 1 \pmod m \Rightarrow \omega_n^{n / 2} \equiv -1 \pmod m \Rightarrow \omega_n^{k + n / 2} \equiv -\omega_n^k \pmod m \Rightarrow \omega_n^k = -\omega_n^{k + n / 2} ωnn​≡ωnn/2​ωnn/2​≡1(modm)⇒ωnn/2​≡−1(modm)⇒ωnk+n/2​≡−ωnk​(modm)⇒ωnk​=−ωnk+n/2​
